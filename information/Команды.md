# Поддерживаемые SQL команды

В данной СУБД реализована поддержка подмножества стандартных SQL-команд для управления базами данных, таблицами и данными.

## Управление базами данных

*   **CREATE DATABASE**
    *   Создает новую базу данных.
    *   Синтаксис: `CREATE DATABASE database_name;`
*   **DROP DATABASE**
    *   Удаляет существующую базу данных.
    *   Синтаксис: `DROP DATABASE database_name;`
*   **USE**
    *   Выбирает базу данных для выполнения последующих операций.
    *   Синтаксис: `USE database_name;`

## Управление таблицами

*   **CREATE TABLE**
    *   Создает новую таблицу в текущей базе данных.
    *   Поддерживает типы `INT` и `STRING`.
    *   Поддерживает модификаторы: `NOT_NULL`, `INDEXED`, `DEFAULT value`.
    *   Синтаксис: `CREATE TABLE table_name (column_name TYPE [MODIFIERS], ...);`
*   **DROP TABLE**
    *   Удаляет таблицу из текущей (или указанной через точку) базы данных.
    *   Синтаксис: `DROP TABLE table_name;` или `DROP TABLE database_name.table_name;`

## Манипуляция данными

*   **INSERT INTO**
    *   Добавляет новую строку в таблицу.
    *   Синтаксис: `INSERT INTO table_name (col1, col2) VALUE (val1, val2);`
*   **UPDATE**
    *   Изменяет существующие данные в таблице.
    *   Синтаксис: `UPDATE table_name SET col1 = val1 WHERE condition;`
*   **DELETE FROM**
    *   Удаляет строки из таблицы.
    *   Синтаксис: `DELETE FROM table_name WHERE condition;`

## Запросы данных

*   **SELECT**
    *   Извлекает данные из таблицы.
    *   Поддерживает: `*`, список столбцов, `AS` для алиасов, агрегатные функции `COUNT`, `SUM`, `AVG`.
    *   Поддерживает `WHERE` с операторами `==`, `!=`, `<`, `<=`, `>`, `>=`, `BETWEEN`, `LIKE`, а также сложные условия с `AND`, `OR` и скобками.
    *   Синтаксис: `SELECT [cols|*] FROM table_name [WHERE condition];`

## Системные команды

*   **REVERT**
    *   Откатывает состояние базы данных к определенному моменту времени, используя историю запросов из логов.
    *   Синтаксис: `REVERT [yyyy.mm.dd-hh:mm:ss.msmsms];` или `REVERT table_name [yyyy.mm.dd-hh:mm:ss.msmsms];`
*   **EXIT / QUIT**
    *   Завершает работу интерактивного режима.
    *   Синтаксис: `EXIT;` или `QUIT;`
