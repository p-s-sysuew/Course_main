# Работа с сервером

СУБД поддерживает клиент-серверную архитектуру, позволяя выполнять SQL-запросы удаленно через HTTP API.

## Запуск сервера

Сервер запускается командой:
```bash
./build/course_server [порт]
```
По умолчанию используется порт 8080.

## API Эндпоинты

*   **GET /health**: Проверка работоспособности сервера.
*   **POST /register**: Регистрация нового пользователя.
*   **POST /login**: Аутентификация и получение JWT токена.
*   **POST /query**: Выполнение SQL-запроса (требует Authorization заголовок с токеном).
*   **GET /metrics**: Получение телеметрии сервера в текстовом формате.
*   **GET /heartbeat**: Проверка состояния системных компонентов.

## Клиентское приложение

Для удобства предусмотрен готовый клиент:
```bash
./build/course_client <host> <port> <sql_file> <user> <password>
```
Он автоматически выполняет авторизацию и отправляет содержимое SQL-файла на сервер.

## Особенности реализации

*   **Безопасность**: Доступ к выполнению запросов защищен с помощью JWT.
*   **Параллелизм**: Для каждой таблицы используется отдельный мьютекс, что позволяет выполнять запросы к разным таблицам одновременно.
*   **Логирование**: Все запросы к серверу логируются в бинарный файл `logs/access.pb`.
