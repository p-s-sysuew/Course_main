# Логирование в СУБД

Система логирования предназначена для аудита всех выполняемых запросов и отслеживания состояния сервера.

## Формат логов

Логи хранятся в бинарном зашифрованном формате с использованием **Google Protocol Buffers (Protobuf)**. Это обеспечивает:
*   Компактность хранения.
*   Высокую скорость записи.
*   Защиту данных от прямого чтения.

Основной файл логов: `logs/access.pb`.

## Структура записи (ProtoLogEntry)

Каждая запись в логе содержит:
*   **client_id**: Идентификатор пользователя или клиента.
*   **handler_id**: Тип обработчика (terminal, script, api).
*   **query_text**: Текст SQL-запроса.
*   **start_time / finish_time**: Время начала и окончания выполнения.
*   **status**: Результат выполнения (OK или ERROR).
*   **message**: Дополнительное сообщение или текст ошибки.

## Просмотр логов

Так как логи бинарные и зашифрованные, для их чтения используется специальная утилита:

```bash
./build/course_logs [путь_к_файлу_логов]
```

По умолчанию утилита ищет файл в `logs/access.pb`. Она расшифровывает записи и выводит их в консоль в человекочитаемом виде.
