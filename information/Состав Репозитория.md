# Состав репозитория

Обзор структуры файлов проекта и их назначения.

## Директории

*   **include/**: Заголовочные файлы (.h).
*   **src/**: Исходный код (.cpp).
*   **scripts/**: SQL-скрипты для демонстрации и нагрузочного тестирования.
*   **information/**: Документация по проекту.

## Основные файлы

### Ядро СУБД
*   `dbms.h/cpp`: Главный класс системы, управляющий базами данных и таблицами.
*   `storage.h/cpp`: Низкоуровневая работа с хранилищем, упаковка полей в length-prefix формат.
*   `table.h/cpp`: Реализация логики работы с отдельной таблицей.
*   `bstarindex.h`, `diskbstarindex.h/cpp`: Реализация B*-дерева для индексации.
*   `stringpool.h/cpp`: Пул строк для оптимизации хранения повторяющихся данных (String Interning).

### Парсинг и выполнение
*   `lexer.h/cpp`: Лексический анализатор (разбиение текста на токены).
*   `parser.h/cpp`: Синтаксический анализатор (построение команд из токенов).
*   `runner.h/cpp`: Исполнитель SQL-команд.

### Сервер и Клиент
*   `server.cpp`: HTTP-сервер на базе Crow.
*   `client.cpp`: Клиентское приложение для отправки запросов.
*   `auth.h/cpp`: Система аутентификации (RBAC, JWT).

### Вспомогательные системы
*   `logger.h/cpp`: Система бинарного логирования запросов.
*   `logviewer.cpp`: Утилита для просмотра зашифрованных логов.
*   `telemetry.h/cpp`: Сбор метрик производительности.
*   `heartbeat.h/cpp`: Мониторинг состояния сервисов.
*   `securefile.h/cpp`: Обёртка для защищенного (зашифрованного) хранения файлов с использованием Protobuf.



# Обзор файлов проекта

| Файл                    | Назначение |
|-------------------------|----------|
| **auth.cpp**            | Система аутентификации и авторизации. Реализует класс `AuthManager`, работу с JWT-токенами, хранение пользователей, регистрацию, логин и проверку прав (RBAC). |
| **client.cpp**          | **Клиентское приложение**. Отправляет HTTP-запросы на сервер (`/login` и `/query`). Поддерживает запуск как с файлом скрипта, так и с прямой SQL-командой. |
| **dbms.cpp**            | Центральный класс СУБД — `DBMS`. Отвечает за работу с базами данных: `CREATE DATABASE`, `DROP DATABASE`, `USE`, диспетчеризацию всех SQL-команд. |
| **diskbstarindex.cpp**  | Реализация **B*-дерева** на диске для индексов. Поддерживает вставку, удаление и поиск по индексированным столбцам. |
| **heartbeat.cpp**       | Мониторинг здоровья других узлов. Проверяет `/health` и может автоматически перезапускать упавшие узлы. |
| **jsonhelper.cpp**      | Вспомогательные функции для преобразования результатов запросов в JSON (используется в SELECT и агрегатных функциях). |
| **lexer.cpp**           | **Лексер** (токенизатор). Разбивает SQL-текст на токены: ключевые слова, идентификаторы, строки, числа и символы. |
| **logger.cpp**          | Система логирования всех выполненных запросов. Записывает информацию в `logs/access.pb`. |
| **logviewer.cpp**       | Отдельная утилита для просмотра логов (`./logviewer`). |
| **main.cpp**            | **Основная программа** в standalone-режиме. Запускает интерактивный или пакетный режим выполнения SQL. |
| **parser.cpp**          | **Парсер SQL**. Преобразует текст в структуры команд (`CreateTableCommand`, `SelectCommand`, `InsertCommand` и т.д.). |
| **runner.cpp**          | Исполнение SQL-текста. Содержит функции `executeText()` и `executeTextAuthorized()` (с проверкой прав пользователя). |
| **securefile.cpp**      | **Защищённое хранение данных**. Реализует шифрование записей (XOR), формат с magic header и работу с tombstone-слоями. |
| **server.cpp**          | **HTTP-сервер** (основное серверное приложение). Обрабатывает запросы `/login`, `/register`, `/query` с помощью библиотеки Crow. |
| **storage.cpp**         | Сериализация и десериализация данных (строки, схемы таблиц, значений) в/из Protobuf. |
| **stringpool.cpp**      | Пул строк (String Pool). Оптимизирует хранение строк путём дедупликации. |
| **table.cpp**           | **Самый важный файл**. Реализует класс `Table` — всю логику работы с таблицами (CREATE, INSERT, UPDATE, DELETE, SELECT, индексы, DEFAULT). |
| **tablelocks.cpp**      | Механизм блокировок таблиц (отдельный mutex для каждой таблицы) для обеспечения thread-safety. |
| **telemetry.cpp**       | Сбор и вывод метрик производительности (RPS, количество ошибок, среднее время выполнения запросов). |
| **utils.cpp**           | Общие вспомогательные функции: `trim`, `splitStatements`, `isValidIdentifier`, `nowText`, `escapeJsonString` и др. |
