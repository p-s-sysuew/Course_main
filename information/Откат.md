# Откат к предыдущим версиям

В данной СУБД механизмы отката реализованы на двух уровнях: на уровне исходного кода и на уровне данных (с ограничениями).

## Откат изменений кода (Git)

Если вам необходимо вернуться к предыдущему состоянию разработки, используйте стандартные команды Git:

1.  **Просмотр истории**:
    ```bash
    git log --oneline
    ```
2.  **Временный переход к версии**:
    ```bash
    git checkout <commit_hash>
    ```
3.  **Полный откат (с потерей незакоммиченных изменений)**:
    ```bash
    git reset --hard <commit_hash>
    ```
4.  **Создание обратного коммита (безопасный способ)**:
    ```bash
    git revert <commit_hash>
    ```

## Откат данных

**Важно**: На текущий момент СУБД не поддерживает SQL-команду `ROLLBACK` или транзакции в классическом понимании ACID. Все изменения (`INSERT`, `UPDATE`, `DELETE`) применяются к файлам данных немедленно.

### Рекомендации по работе с данными:
1.  **Бэкапы**: Перед выполнением опасных операций копируйте содержимое директории `data/`.
2.  **Восстановление**: Для восстановления базы данных просто замените текущую директорию `data/` на ранее сохраненную копию.
3.  **Логи**: Используйте `logs/access.pb` для анализа того, какие команды привели к нежелательному состоянию данных.
